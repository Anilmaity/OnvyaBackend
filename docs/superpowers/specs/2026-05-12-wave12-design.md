# Wave 1+2 Backend & Console Design

**Date:** 2026-05-12
**Scope:** Five new Django apps (`scheduling`, `documents`, `invoicing`, `training`, plus `time-adjustments` co-located in `scheduling`) and end-to-end wiring of the matching console pages so they show live data instead of demo placeholders.

## 1. App boundaries

Four new apps under `apps/`, following the existing convention (`models.py`, `schema.py`, `services.py`, `selectors.py`, `tests/`, `migrations/`, `admin.py`):

| App | Owns | Depends on |
|---|---|---|
| `apps.scheduling` | `Shift`, `TimeAdjustment` | `drivers`, `agencies` (Depot), `accounts` |
| `apps.documents` | `DriverDocument` (metadata only) | `drivers` |
| `apps.invoicing` | `Invoice`, `InvoiceLineItem`, `InvoiceCounter` | `drivers`, `scheduling` (read-only) |
| `apps.training` | `Course`, `Completion` | `drivers` |

All models inherit `AgencyScopedModel`. Tenant context is set by the existing `AgencyContextMiddleware`. GraphQL `Query`/`Mutation` classes are composed into `config/schema.py`.

**Permissions** added to `apps/accounts/data/permission_catalogue.py`:
- `scheduling.read`, `scheduling.create`, `scheduling.update`, `scheduling.approve_adjustment`
- `documents.read`, `documents.write`
- `invoicing.read`, `invoicing.create`, `invoicing.update`
- `training.read`, `training.write`

Role grants:
- **Super Admin** — all of the above.
- **Fleet Manager** — `scheduling.*`, `invoicing.read|create|update`, `documents.read`, `training.read`.
- **Compliance Officer** — `documents.*`, `training.*`, `scheduling.read`.
- **Finance Admin** — `invoicing.*`, `scheduling.read`.
- **Driver** — no console permissions added this wave (mobile-side read-only of own data is out of scope).

**Cross-app contracts:** Only `invoicing → scheduling` (read). `apps/invoicing/services.py::InvoiceService.generate_draft` calls `apps/scheduling/selectors.py::list_shifts_for_billing(driver, start, end)`. No reverse dependency.

## 2. `apps.scheduling`

### Models

```python
class Shift(AgencyScopedModel):
    class Status(TextChoices):
        SCHEDULED = "SCHEDULED"
        IN_PROGRESS = "IN_PROGRESS"
        COMPLETED = "COMPLETED"
        MISSED = "MISSED"
        CANCELLED = "CANCELLED"
    driver = FK("drivers.Driver", on_delete=PROTECT, related_name="shifts")
    depot = FK("agencies.Depot", null=True, blank=True, on_delete=SET_NULL)
    start = DateTimeField()
    end = DateTimeField()
    actual_start = DateTimeField(null=True, blank=True)
    actual_end = DateTimeField(null=True, blank=True)
    status = CharField(max_length=16, choices=Status.choices, default=SCHEDULED)
    notes = TextField(blank=True, default="")
    billable_hours = DecimalField(max_digits=5, decimal_places=2, null=True, blank=True)
    hourly_rate = DecimalField(max_digits=6, decimal_places=2, null=True, blank=True)
    template_id = UUIDField(null=True, blank=True, db_index=True)  # forward-compat
    class Meta(AgencyScopedModel.Meta):
        indexes = AgencyScopedModel.Meta.indexes + [
            Index(fields=["agency","driver","start"]),
            Index(fields=["agency","status","start"]),
        ]

class TimeAdjustment(AgencyScopedModel):
    class State(TextChoices):
        PENDING = "PENDING"; APPROVED = "APPROVED"; REJECTED = "REJECTED"
    shift = FK(Shift, on_delete=CASCADE, related_name="adjustments")
    requested_by = FK("accounts.AgencyUser", on_delete=SET_NULL, null=True, related_name="requested_adjustments")
    decided_by = FK("accounts.AgencyUser", on_delete=SET_NULL, null=True, related_name="decided_adjustments")
    proposed_start = DateTimeField()
    proposed_end = DateTimeField()
    reason = TextField()
    state = CharField(max_length=16, choices=State.choices, default=PENDING)
    decided_at = DateTimeField(null=True, blank=True)
    decision_note = TextField(blank=True, default="")
```

### Service rules

`ShiftService`:
- `create(driver, start, end, depot=None, notes="") -> Shift` — validates `end > start`, status SCHEDULED.
- `update(shift, **fields)` — only allowed when status ∈ {SCHEDULED, IN_PROGRESS}; raises `IllegalTransition` otherwise.
- `start(shift, at)` — SCHEDULED → IN_PROGRESS, sets `actual_start = at`.
- `complete(shift, actual_start=None, actual_end=None)` — IN_PROGRESS → COMPLETED. Sets `actual_*` defaulting to `start`/`end`. Computes `billable_hours = round_quarter((actual_end - actual_start).total_seconds()/3600)`, freezes `hourly_rate` from constant `DEFAULT_HOURLY_RATE` (config). Idempotent if already COMPLETED.
- `cancel(shift)` — SCHEDULED → CANCELLED only.
- `mark_missed(shift)` — for cron sweep of past SCHEDULED shifts.

`TimeAdjustmentService`:
- `request(shift, user, proposed_start, proposed_end, reason) -> TimeAdjustment` — blocks if `_shift_locked_by_invoice(shift)`.
- `approve(adj, user, decision_note="")` — PENDING only. Applies proposed times to `shift.actual_start/end`, re-calls `ShiftService.complete(...)` to recompute `billable_hours` if shift is COMPLETED. Sets state APPROVED, `decided_by`, `decided_at`.
- `reject(adj, user, decision_note)` — PENDING only. No shift mutation.
- `_shift_locked_by_invoice(shift)` — true if any non-VOID, non-DRAFT `InvoiceLineItem` references this shift.

`round_quarter(hours)` → `Decimal` rounded to nearest 0.25.

### GraphQL surface

Queries:
- `shifts(filter: ShiftFilter): [ShiftType!]!`
- `shift(id: ID!): ShiftType`
- `timeAdjustments(filter: TimeAdjustmentFilter): [TimeAdjustmentType!]!`
- `timeAdjustment(id: ID!): TimeAdjustmentType`

`ShiftFilter`: `driverId`, `depotId`, `status`, `startAfter`, `startBefore`.
`TimeAdjustmentFilter`: `shiftId`, `state`.

`ShiftType` exposes: id, driver, depot, start, end, actual_start, actual_end, status, notes, billable_hours, hourly_rate, adjustments (list).

Mutations (all return `MutationResult`):
- `createShift(input: CreateShiftInput!)`
- `updateShift(id: ID!, input: UpdateShiftInput!)`
- `cancelShift(id: ID!)`
- `completeShift(id: ID!, actualStart: DateTime, actualEnd: DateTime)`
- `requestTimeAdjustment(input: RequestTimeAdjustmentInput!)`
- `approveTimeAdjustment(id: ID!, decisionNote: String)`
- `rejectTimeAdjustment(id: ID!, decisionNote: String!)`

```python
class DriverDocument(AgencyScopedModel):
    class Kind(TextChoices):
        DRIVING_LICENCE = "DRIVING_LICENCE"
        RIGHT_TO_WORK   = "RIGHT_TO_WORK"
        INSURANCE       = "INSURANCE"
        MOT             = "MOT"
        DBS             = "DBS"
        CPC             = "CPC"
    class Status(TextChoices):
        VALID = "VALID"; EXPIRING = "EXPIRING"; EXPIRED = "EXPIRED"; MISSING = "MISSING"
    driver = FK("drivers.Driver", on_delete=CASCADE, related_name="documents")
    kind = CharField(max_length=32, choices=Kind.choices)
    reference = CharField(max_length=128, blank=True, default="")
    issued_on = DateField(null=True, blank=True)
    expires_on = DateField(null=True, blank=True)
    status = CharField(max_length=16, choices=Status.choices, default=MISSING)
    notes = TextField(blank=True, default="")
    class Meta(AgencyScopedModel.Meta):
        constraints = [UniqueConstraint(fields=["agency","driver","kind"], name="document_unique_kind_per_driver")]
        indexes = AgencyScopedModel.Meta.indexes + [Index(fields=["agency","status","expires_on"])]
```

### Service

`DocumentService.upsert(driver, kind, **fields) -> DriverDocument` — `update_or_create` by `(driver, kind)`, calls `_recompute_status(doc)` before save:
- `expires_on is None` → MISSING.
- `expires_on < today` → EXPIRED.
- `expires_on <= today + 30d` → EXPIRING.
- else → VALID.

`DocumentService.recompute_all(agency=None)` — iterates all rows, calls `_recompute_status`, saves only if changed. Exposed as `manage.py recompute_documents`.

### GraphQL

- `driverDocuments(filter: DriverDocumentFilter): [DriverDocumentType!]!` (`driverId`, `kind`, `status`, `expiresBefore`)
- `driverDocument(id: ID!): DriverDocumentType`
- Mutations: `upsertDriverDocument(input)`, `deleteDriverDocument(id)`.

## 4. `apps.invoicing`

### Models

```python
class Invoice(AgencyScopedModel):
    class Status(TextChoices):
        DRAFT = "DRAFT"; ISSUED = "ISSUED"; PAID = "PAID"; VOID = "VOID"
    driver = FK("drivers.Driver", on_delete=PROTECT, related_name="invoices")
    number = CharField(max_length=32)  # placeholder while DRAFT
    period_start = DateField()
    period_end = DateField()
    issued_on = DateField(null=True, blank=True)
    due_on = DateField(null=True, blank=True)
    paid_on = DateField(null=True, blank=True)
    subtotal = DecimalField(max_digits=10, decimal_places=2, default=0)
    vat = DecimalField(max_digits=10, decimal_places=2, default=0)
    total = DecimalField(max_digits=10, decimal_places=2, default=0)
    status = CharField(max_length=16, choices=Status.choices, default=DRAFT)
    notes = TextField(blank=True, default="")
    class Meta(AgencyScopedModel.Meta):
        constraints = [UniqueConstraint(fields=["agency","number"], name="invoice_unique_number_per_agency")]
        indexes = AgencyScopedModel.Meta.indexes + [Index(fields=["agency","status","period_end"])]

class InvoiceLineItem(AgencyScopedModel):
    invoice = FK(Invoice, on_delete=CASCADE, related_name="line_items")
    shift = FK("scheduling.Shift", null=True, blank=True, on_delete=SET_NULL, related_name="invoice_lines")
    description = CharField(max_length=255)
    quantity = DecimalField(max_digits=8, decimal_places=2)
    unit_price = DecimalField(max_digits=8, decimal_places=2)
    amount = DecimalField(max_digits=10, decimal_places=2)

class InvoiceCounter(AgencyScopedModel):
    year = PositiveIntegerField()
    last_number = PositiveIntegerField(default=0)
    class Meta(AgencyScopedModel.Meta):
        constraints = [UniqueConstraint(fields=["agency","year"], name="invoice_counter_unique_per_year")]
```

### Service

`InvoiceService.generate_draft(driver, period_start, period_end) -> Invoice | ValidationError`:
- Eligible shifts: `Shift.objects.filter(agency=agency, driver=driver, status=COMPLETED, actual_end__date__range=(period_start, period_end)).exclude(invoice_lines__invoice__status__in=[DRAFT, ISSUED, PAID])`.
- If empty → `ValidationError("no_shifts")`.
- Create Invoice with status DRAFT, `number = f"DRAFT-{uuid4().hex[:8].upper()}"`.
- One `InvoiceLineItem` per shift: `description = f"Shift {start_date}"`, `quantity = shift.billable_hours`, `unit_price = shift.hourly_rate`, `amount = q*p`.
- Compute totals: `subtotal = sum(amount)`, `vat = subtotal * INVOICE_VAT_RATE` (config, default 0.20), `total = subtotal + vat`. Persisted on Invoice.

`InvoiceService.issue(invoice) -> Invoice`:
- DRAFT only.
- In `transaction.atomic`: `select_for_update` the `InvoiceCounter` row for `(agency, current_year)`, increment `last_number`, set invoice `number = f"INV-{year}-{n:04d}"`, status ISSUED, `issued_on = today`, `due_on = today + INVOICE_DUE_DAYS` (default 14).

`InvoiceService.update_draft(invoice, line_items)`:
- DRAFT only. Replaces line items, recomputes totals.

`InvoiceService.mark_paid(invoice, paid_on)` — ISSUED → PAID.
`InvoiceService.void(invoice, reason)` — DRAFT or ISSUED → VOID.

After ISSUED: edits to line items, totals, or period dates raise `IllegalTransition` at service layer. Permissions can't bypass.

### GraphQL

- `invoices(filter: InvoiceFilter): [InvoiceType!]!` (`driverId`, `status`, `periodStartAfter`, `periodEndBefore`)
- `invoice(id: ID!): InvoiceType` (includes `lineItems`)
- Mutations: `generateDraftInvoice(input)`, `updateDraftInvoice(id, input)`, `issueInvoice(id)`, `markInvoicePaid(id, paidOn)`, `voidInvoice(id, reason)`.

```python
    is_required = BooleanField(default=False)

class Completion(AgencyScopedModel):
    class Status(TextChoices):
        VALID = "VALID"; EXPIRING = "EXPIRING"; EXPIRED = "EXPIRED"
    driver = FK("drivers.Driver", on_delete=CASCADE, related_name="training")
    course = FK(Course, on_delete=CASCADE, related_name="completions")
    completed_on = DateField()
    expires_on = DateField(null=True, blank=True)

```

### Service

`CompletionService.upsert(driver, course, completed_on, certificate_reference="", notes="")`:
- Computes `expires_on = completed_on + relativedelta(months=course.validity_months)` if `validity_months` is set, else `None`.
- Computes `status` (same rules as documents; if `expires_on is None`, VALID forever).
- `update_or_create` on `(driver, course)`.

`CompletionService.recompute_all(agency=None)` + `manage.py recompute_training`.

`TrainingSelectors.missing_required_courses(driver) -> QuerySet[Course]`:
- `Course.objects.filter(agency=driver.agency, is_required=True).exclude(completions__driver=driver)`.

### GraphQL

- `courses: [CourseType!]!`
- `completions(filter: CompletionFilter): [CompletionType!]!`
- `driverTraining(driverId: ID!): DriverTrainingType` where:
  - `completed: [CompletionType!]!`
  - `missing: [CourseType!]!`
- Mutations: `upsertCourse(input)`, `deleteCourse(id)`, `recordCompletion(input)`, `deleteCompletion(id)`.

## 6. Seed data (`seed_demo` extended)

Idempotent additions on top of existing seed:

- **Scheduling:** for each of 6 active drivers, 14 days of past shifts + 14 days of future shifts, weekdays only, planned 09:00–17:00 with ±15min jitter on actuals. ~80% past shifts COMPLETED, 10% MISSED, 10% CANCELLED. Hourly rate £15. Future shifts SCHEDULED.
- **Time-adjustments:** 3 PENDING (recent COMPLETED shifts, reasons "forgot to clock out", "traffic delay", "extended customer interaction"), 1 APPROVED, 1 REJECTED.
- **Documents:** every driver gets 6 documents (one per `Kind`). Mix tuned so:
  - At least 2 EXPIRING (≤ 30d).
  - 1 EXPIRED (DRIVING_LICENCE on a designated driver — feeds dashboard alert).
  - 1 MISSING (DBS on a designated driver).
  - Remainder VALID.
- **Invoicing:** for the 3 highest-shift-count drivers, generate DRAFT invoices for the past 14-day window via real `InvoiceService.generate_draft`. One DRAFT → `issue` → `mark_paid`. One DRAFT → `issue`. One stays DRAFT.
- **Training:** 5 Courses — "Safety Induction" (required, 12mo), "Defensive Driving" (required, 24mo), "Manual Handling" (required, 36mo), "Customer Service" (optional, null), "First Aid" (optional, 36mo). Completions distributed so:
  - 2 drivers fully compliant on all required.
  - 2 drivers with at least one EXPIRING.
  - 1 driver with one EXPIRED.
  - 1 driver missing one required course.

All seed lookups use `get_or_create` / `update_or_create` on natural keys.

## 7. Console wiring

| Page | Action |

## 8. Testing

Per app, in `apps/<name>/tests/`:
- `test_services.py` — state-machine transitions, computed fields, illegal-transition errors, idempotency of upsert/seed.
- `test_schema.py` — resolver-level filter, auth, shape. Uses `graphene.test.Client` and the existing test agency context.

Critical end-to-end tests:
- `scheduling.test_complete_shift_locks_billable` — `complete(...)` freezes `billable_hours`/`hourly_rate`.
- `scheduling.test_adjustment_blocked_when_invoice_issued` — once an invoice referencing the shift is ISSUED, adjustments raise `IllegalAdjustment`.
- `invoicing.test_generate_draft_uses_only_completed` — DRAFT excludes SCHEDULED/IN_PROGRESS/CANCELLED/MISSED.
- `invoicing.test_generate_draft_excludes_already_invoiced_shifts` — second generation for same period returns `no_shifts`.
- `invoicing.test_issue_allocates_monotonic_number` — concurrent calls don't collide (covered by `select_for_update`).
- `documents.test_status_recompute_on_save` — VALID → EXPIRING when expires_on moved inside 30d window.
- `documents.test_recompute_all_idempotent` — second run mutates no rows.
- `training.test_missing_required_excludes_completed` — `missing_required_courses` returns only non-completed required courses.
- `training.test_completion_expiry_computed` — `expires_on = completed_on + validity_months`.

Console: `npx tsc --noEmit` and `npm run build` must pass after each page is wired. No new JS unit tests this wave (no infra yet).
