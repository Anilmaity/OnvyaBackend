# Wave 1+2 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship `scheduling`, `documents`, `training`, `invoicing` Django apps end-to-end (models, services, GraphQL, tests, seed) and wire the matching `/scheduling`, `/time-adjustments/[id]`, `/documents`, `/invoicing`, `/training` console pages to live data.

**Architecture:** Each domain is its own `apps.<name>` Django app following the established convention (`models.py`, `services.py`, `selectors.py`, `schema.py`, `admin.py`, `migrations/`, `tests/`). All models inherit `AgencyScopedModel`. GraphQL types composed into `config/schema.py`. Permissions added to `apps/accounts/data/permission_catalogue.py`. Console: each page replaces local `lib/data/*` imports with Apollo `useQuery`/`useMutation` and drops its `DemoDataBadge`.

**Tech Stack:** Django 5, graphene-django, PostgreSQL, Next.js App Router, Apollo Client, Zustand.

**Spec:** `docs/superpowers/specs/2026-05-12-wave12-design.md`
